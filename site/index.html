How It Works

Running an audit in six steps

From credential setup to a reviewed, exported report — the entire process takes minutes.

🔐
1

Configure OAuth Credentials

Create a Client Credentials OAuth client in your Genesys Cloud Admin portal. Store the Client ID and Secret using .NET user secrets or environment variables — they are never written to disk or committed to source control.

🎯
2

Select Audit Scope

Choose which audit paths to run: extensions, groups, queues, flows, DIDs, inactive users, or event log exports. Toggle whether inactive users should be included in the scan.

▶️
3

Run the Audit

Click Start. The auditor fetches all paginated data from the Genesys Cloud API, normalizes extensions to canonical form, and cross-references users and telephony assignments. A live progress bar and status log keep you informed.

📑
4

Review Findings

Results appear in tabbed sheets — one per finding type. Critical findings (duplicate extensions, ownership mismatches) are highlighted at the top. Drill into each group to see which users or assignments are involved.

📤
5

Export to Excel

Click Export Last Report… to save a multi-sheet Excel workbook. Each sheet maps to one audit check, with a Summary sheet showing finding counts and severity across the full run.

✅
6

Act on Recommendations

Each finding includes context for remediation: which user, which extension, and what the mismatch is. Clear the stale value, reassign the DID, republish the flow — then re-run the audit to confirm zero findings.
